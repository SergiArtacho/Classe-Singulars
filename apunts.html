L'estructura d'HTML

En el moment de crear el document base, l'estructura d'HTML és la següent:

    <!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <title>Document</title>
        <style></style>
    </head>
    <body>
        
    </body>
    </html>

El text comença indicant-nos el format <!DOCTYPE html> , seguidament hem d'indicar la llengua en que escriurem la nostre web , <html lang="en">.

L'element <head> és un contenidor per a metadades (dades sobre dades) i se situa entre l'etiqueta html i l'etiqueta <body>.Les metadades HTML són dades sobre el document HTML. Les metadades no es mostren.Normalment, les metadades defineixen el títol del document, el conjunt de caràcters, els estils, els scripts i altra informació meta. L'element <body> conté tot el contingut d'un document HTML, com ara encapçalaments, paràgrafs, imatges, hiperenllaços, taules, llistes, etc. Nota: Només hi pot haver un element <body> en un document HTML. Per configurar l'estètica en aquest cas amb el CSS, L’atribut <style> s’utilitza per afegir estils a un element, com ara color, tipus de lletra, mida i molt més.

Si volem fer comentaris sobre el codi sense que s'executin, a Html, hem de fer <!----> i a CSS /* Comentari */

Display

La propietat de visualització especifica el comportament de visualització (el tipus de quadre de representació) d'un element. En HTML, el valor de propietat de visualització per defecte es pren de les especificacions HTML o del full d'estil predeterminat del navegador / usuari. El valor per defecte a XML és en línia, inclosos els elements SVG.

  • Inline––––> Mostra un element com a element en línia. Les propietats d’alçada i amplada no tindran cap efecte

  • Block––––> Mostra un element com a element de bloc. Comença per una línia nova i ocupa tota l'amplada

  • Flex–––> Mostra un element com a contenidor flex a nivell de bloc

  • Gird––––> Mostra un element com a contenidor de quadrícula a nivell de bloc.

  • Inline-block––––> Mostra un element com a contenidor de blocs a nivell de línia. L'element en sí té el format d'un element en línia, però podeu aplicar valors d'alçada i amplada

  • Inline-flex––––––> Mostra un element com a contenidor flexible a nivell de línia.

  • Inline-gird––––––> Mostra un element com a contenidor de quadrícula en línia

  • Display-none: Es pot ocultar un element configurant la propietat de visualització a cap. L'element quedarà amagat i la pàgina es mostrarà com si l'element no hi fos

Float

La propietat CSS float situa un element a la banda esquerra o dreta del seu contenidor, permetent als elements de text i en línia aparèixer al seu costat. L'element és remogut del flux normal de la pàgina, tot i que encara segueix sent part d'ell (a diferència de l'posicionament absolut).

Git Hub

GIT: Git és un sistema de control de versions de codi obert. Els desenvolupadors apliquen canvis en els seus codis tot el temps. Primer per a les versions beta, després vénen les versions estables o definitives, i també després de l'llançament de l'aplicació o lloc web que estan alternant.

GitHub és un aplicació web per a control de canvis i documentació, que funciona alhora com hosting de codi i xarxa social entre desenvolupadors.

Funcions Git Hub:

  • Funciona a mode de repositori

  • Control de canvis

  • Documentació de projectes

  • És un sistema segur

  • És pot utilitzar directament des de Visual Studio Code

Comandos esencials de Git:

  • git init–––> Crea el repositori Git en el local per que puguis començar a treballar.

  • git add–––> Utilitza git add per afegir arxius a l'índex de el projecte. Aquest comando s'utilitza afegint a la fin el nom de l'arxiu amb un espai a la fi de la comanda com ho veus en l'exemple sota.

  • git commit––––> Guarda els canvis realitzats amb git commit. Al desar podràs especificar un missatge per identificar el canvi que vas realitzar o afegir un comentari. Exemple: git commit -m "mensaje de referencia"

  • git branch––––> Aquest comando es fa servir per llistar, crear o esborrar branques. Per llistar totes les branques utilitza:

  • git stash––––> S'utilitza per guardar canvis temporals

  • git log––––> Pots veure una llista dels commits realitzats.

  • git tag–––> S'utilitza per etiquetar commits especifics, per exemple: git tag 1.0.1 <id de commit>

  • Git diff––––> Llista de conflictes existents

  • git merge–––> per fusionar un branch actiu amb un altre. Ex: git merge <nombre del branch>

  • git remote add ––––> Agrega un repositiori remot. Si fem get remote add . Ens ho agrega tot, el punt va amb un espai.

  • git clone–––> S'utilitza per revisar repositoris.

  • git push––––> Envia els canvis que s'han fet.

  • git pull––––> Fusiona tots els canvis que s'han fet en el repositori local.

  • git fetch––––> Busca tots els recursos d'un repositori remot que actualment no están en el directori local.

Formularios

S'utilitza un formulari HTML per recollir les dades de l'usuari. L’entrada de l’usuari s’envia a un servidor per processar-la.

Elements:

  • L'element <form>: s'utilitza per crear un formulari HTML per a l'entrada de l'usuari: L’element <form> és un contenidor per a diferents tipus d’elements d’entrada, com ara: camps de text, caselles de selecció, botons d’opció, botons d’enviament, etc.

  • L'element HTML <input> és l'element de formulari més utilitzat. Un element <input> es pot mostrar de moltes maneres, segons l'atribut type. Aquests són alguns exemples:

    • <input type = "text"> Mostra un camp d'entrada de text d'una sola línia

    • <input type = "radio"> Mostra un botó d'opció (per seleccionar una de les moltes opcions)

    • <input type = "checkbox"> Mostra una casella de selecció (per seleccionar zero o més de moltes opcions)

    • <input type = "submit"> Mostra un botó d'enviament (per enviar el formulari)

    • <input type = "button"> Mostra un botó on es pot fer clic

  • El <input type = "text"> defineix un camp d'entrada d'una sola línia per a l'entrada de text.

  • The <label> tag defines a label for many form elements.Is useful for screen-reader users, because the screen-reader will read out loud the label when the user focus on the input element. The <label> element also help users who have difficulty clicking on very small regions (such as radio buttons or checkboxes) - because when the user clicks the text within the <label> element, it toggles the radio button/checkbox. The for attribute of the <label> tag should be equal to the id attribute of the <input> element to bind them together.

  • El <input type = "radio"> defineix un botó d'opció. Els botons d'opció permeten a l'usuari seleccionar UNA d'un nombre limitat d'opcions.

  • La <input type = "checkbox"> defineix una casella de selecció. Les caselles de selecció permeten a un usuari seleccionar ZERO o MES opcions d'un nombre limitat d'opcions.

  • El <input type = "submit"> defineix un botó per enviar les dades del formulari a un gestor de formularis. El gestor de formularis sol ser un fitxer del servidor amb un script per processar les dades d’entrada. El gestor de formularis s’especifica a l’atribut d’acció del formulari.

    • Cal tenir en compte que cada camp d’entrada ha de tenir un atribut de nom per enviar-lo.Si s'omet l'atribut name, el valor del camp d'entrada no s'enviarà en absolut.

KEYFRAMES

La regla @keyframes especifica el codi d'animació. L'animació es crea canviant gradualment d'un conjunt d'estils CSS a un altre. Durant l'animació, podeu canviar el conjunt d'estils CSS moltes vegades. Especifiqueu quan es produirà el canvi d'estil en percentatge o amb les paraules clau "from" i "to", que és igual al 0% i al 100%. El 0% és el començament de l'animació, el 100% és quan l'animació es completa. Consell: per obtenir el millor suport per al navegador, sempre heu de definir els selectors del 0% i del 100%. Nota: Utilitzeu les propietats de l'animació per controlar l'aparença de l'animació i també per vincular l'animació als selectors.

Valors

  • animationname -----> Obligatori. Defineix el nom de l'animació.

  • keyframes-selector––––> Obligatori. Percentatge de la durada de l'animació. Valors legals:0-100%. From (igual al 0%) To (igual que el 100%)

  • css-styles–––––> Obligatori. Una o més propietats d'estil CSS legals

Mediaqueries

Utilitza la regla @media per incloure un bloc de propietats CSS només si es compleix una determinada condició.

Mobile First

Mobile First significa dissenyar per a mòbils abans de dissenyar per a ordinadors o qualsevol altre dispositiu (això farà que la pàgina es mostri més ràpidament en dispositius més petits). Això significa que hem de fer alguns canvis al nostre CSS. En lloc de canviar els estils quan l'amplada sigui inferior a 768 px, hauríem de canviar el disseny quan l'amplada sigui superior a 768 px. Això farà que el nostre disseny mòbil sigui primer.

Punts d'interrupció del dispositiu típics

Hi ha un munt de pantalles i dispositius amb diferents alçades i amplades, de manera que és difícil crear un punt d’interrupció exacte per a cada dispositiu. Per simplificar les coses, podem orientar cinc grups: 



        /* Extra small devices (phones, 600px and down) */
        @media only screen and (max-width: 600px) {...} 

        /* Small devices (portrait tablets and large phones, 600px and up) */
        @media only screen and (min-width: 600px) {...} 

        /* Medium devices (landscape tablets, 768px and up) */
        @media only screen and (min-width: 768px) {...} 

        /* Large devices (laptops/desktops, 992px and up) */
        @media only screen and (min-width: 992px) {...} 

        /* Extra large devices (large laptops and desktops, 1200px and up) */
        @media only screen and (min-width: 1200px) {...}

Bootstrap

Bootstrap és un framework CSS i Javascript dissenyat per a la creació d'interfícies netes i amb un disseny responsive. A més, ofereix un ampli ventall d'eines i funcions, de manera que els usuaris poden crear pràcticament qualsevol tipus de lloc web fent ús dels mateixos.

Es basa en una estructura dividida en un total de 12 columnes que els desenvolupadors poden gestionar en funció de les seves necessitats i preferències, en funció de quatre mides de dispositiu.

El responsive de Bootstrap posa el disseny per a Mobile First. No tots els llocs web té aquesta prioritat.

El principal avantatge és la gran documentació, plantilles i desenvolupaments de components que podem utilitzar gratis.

Exemple de Bootstrap:


                <!DOCTYPE html>
                <html lang="en">
                <head>
                <title>Bootstrap Example</title>
                <meta charset="utf-8">
                <meta name="viewport" content="width=device-width, initial-scale=1">
                <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.4.1/css/bootstrap.min.css">
                <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
                <script src="https://maxcdn.bootstrapcdn.com/bootstrap/3.4.1/js/bootstrap.min.js"></script>
                </head>
                <body>

                <div class="jumbotron text-center">
                <h1>My First Bootstrap Page</h1>
                <p>Resize this responsive page to see the effect!</p> 
                </div>

JAVASCRIPT

Javascript és l'únic llenguatge de programació que funciona en els navegadors de forma nativa (llenguatge interpretat sense necessitat de compilació). Per tant s'utilitza com a complement d'HTML i CSS per crear pàgines webs.

Javascript és el llenguatge de programació encarregat de dotar de major interactivitat i dinamisme a les pàgines web. Quan estigui habilitat s'executa al navegador, no necessita d'un compilador. El navegador llegeix directament el codi, sense necessitat de tercers. Per tant, se li reconeix com un dels tres llenguatges nadius del web al costat de HTML (contingut i la seva estructura) i a CSS (disseny de l'contingut i la seva estructura). Compte, no confunidr Javascript amb Java.

Per què serveix?

Amb aquest llenguatge de programació de la banda de el client (no al servidor) podem crear efectes i animacions sense cap interacció, o responent a esdeveniments causats pel propi usuari tal com els botons premuts i modificacions de DOM (document object model). Per tant, res té a veure amb el llenguatge de programació Java, ja que la seva principal funció és ajudar a crear pàgines webs dinàmiques.

El codi de programació de JavaScript s'executa en els navegadors, ja siguin d'escriptori o mòbils, ja siguin Android o Iphone. Serveix per exactament el mateix, és igual al tipus de dispositiu que s'executi el navegador.

Javascript és capaç de detectar errors en formularis, de crear bonics sliders que s'adaptin a qualsevol pantalla, de fer càlculs matemàtics de forma eficient, de modificar elements d'una pàgina web de manera senzilla. Però també Javascript és l'encarregat de que existeixin eines com Google Analytics, Google Tag Manager, Facebook Pixel i tantes altres, que són clars exemples de JavaScript.

Operacions Aritmetiques

Els operadors aritmètics prenen valors numèrics (poden ser tant literals com ser variables) com a operands seus i retornen un valor numèric únic. Els operadors aritmètics estàndards són la suma (+), la resta (-), la multiplicació (*), i la divisió (/).

També trobem els següents:

Mòdul (%)

L'operador mòdul retorna el mòdul del primer operand amb el segon, això és, var1 modulo var2 en la sentència prèvia, on var1 i var2 són variables. La funció mòdul és la resta entera de dividir var1 per var2


        12 % 5 // 2
        -1 % 2 // -1

Increment (++)

L'operador increment incrementa (afegeix un) al seu operand i retorna un valor.

Emprat com a sufix, és a dir, amb l'operador després de l'operand (per exemple: x++), retorna el valor de l'operand abans d'incrementar-lo.

Emprat com a prefix, és a dir, amb l'operador precedint l'operand (per exemple: ++x), retorna el valor de l'operand després d'incrementar-lo.


        // Sufix
        var x = 3;
        y = x++; // y = 3, x = 4

        // Prefix
        var a = 2;
        b = ++a; // a = 3, b = 3

Decrement (--)

L'operador decrement decrementa (resta un) al seu operand i retorna el seu valor.

Emprat com a sufix, és a dir, amb l'operador després de l'operand (per exemple: x--), retorna el valor de l'operand abans de decrementar-lo.

Emprat com a prefix, és a dir, amb l'operador precedint l'operand (per exemple: --x), retorna el valor de l'operand després de decrementar-lo.


        // Sufix
        var x = 3;
        y = x--; // y = 3, x = 2
            
        // Prefix
        var a = 2;
        b = --a; // a = 1, b = 1

Negació unària (-)

L'operador de negació unària precedeix el seu operand i el nega.


        var x = 3;
        y = -x; // y = -3, x = 3

Condicionals: if...else

La sentència if executa una sentència si una condició especificada és certa. Si la condició és falsa, opcionalment s'executa una declaració diferent


        if (condició)
        sentència1
        [else
        sentència2]

Condició

Una expressió que esdevé o certa o falsa.

sentència 1

És la sentència que s'executa si la condició esdevé certa. Pot ser qualsevol declaració, incloent més sentències if aniuades. Per executar múltimples sentències, es fa servir una sentència block ({ ... }) per agrupar aquestes sentències, si no es vol executar cap sentència, es fa servir una sentència buida.

sentència 2

És la sentència que s'executa si la condició esdevé certa. Pot ser qualsevol declaració, incloent més sentències if aniuades. Per executar múltimples sentències, es fa servir una sentència block ({ ... }) per agrupar aquestes sentències, si no es vol executar cap sentència, es fa servir una sentència buida.

Per executar múltiples sentències dins una clàusula, utilitzeu una sentència block ({ ... }) per agrupar aquestes sentències. En general, és una bona pràctica utilitzar sempre sentències de block, especialment en codi que inclogui sentències if aniuades:


        if (condició) {
            sentència1
        } else {
            sentència2
             }

Condicional Switch

La declaración switch evalúa una expresión, comparando el valor de esa expresión con una instancia case, y ejecuta declaraciones asociadas a ese case, así como las declaraciones en los case que siguen.

expresión: És una expressió que és comparada amb el valor de cada instància case.

case valorN: Una instància casi valorN és usada per a ser comparada amb l'expressió. Si l'expressió coincideix amb el valorN, les declaracions dins de la instància casi s'executen fins que es trobi el final de la declaració switch o fins a trobar una interrupció break.

default: Una instància default, quan és declarada, és executada si el valor de l'expressió no coincideix amb qualsevol de les altres instàncies casi valorN.

Si ocorre una coincidència, el programa executa les declaracions associades corresponents. Si l'expressió coincideix amb múltiples entrades, la primera serà la seleccionada, fins i tot si les majúscules són tingudes en compte.

Exemple:


            switch (expr) {
                case 'Naranjas':
                  console.log('El kilogramo de naranjas cuesta $0.59.');
                  break;
                case 'Manzanas':
                  console.log('El kilogramo de manzanas cuesta $0.32.');
                  break;
                case 'Platanos':
                  console.log('El kilogramo de platanos cuesta $0.48.');
                  break;
                case 'Cerezas':
                  console.log('El kilogramo de cerezas cuesta $3.00.');
                  break;
                case 'Mangos':
                case 'Papayas':
                  console.log('El kilogramo de mangos y papayas cuesta $2.79.');
                  break;
                default:
                  console.log('Lo lamentamos, por el momento no disponemos de ' + expr + '.');
              }
              
              console.log("¿Hay algo más que te quisiera consultar?");

Funcions

Les funcions són un dels blocs de construcció fonamentals en JavaScript. Una funció en JavaScript és similar a un procediment - un conjunt d'instruccions que realitza una tasca o calcula un valor, però perquè un procediment qualifiqui com a funció, ha de prendre alguna entrada i retornar una sortida on hi ha alguna relació òbvia entre l'entrada i la sortida. Per utilitzar una funció, has de definir-la en algun lloc de l'àmbit des del que vols cridar-la.

Definir funcions

Una definició de funció (també anomenada declaració de funció o expressió de funció) consta de la paraula clau function, seguida de:

El nom de la funció.

Una llista de paràmetres de la funció, entre parèntesis i separats per comes.

Les declaracions de JavaScript que defineixen la funció, tancades entre claus, {...}.


        Exemple:
        function square(number) {
            return number * number;
              }

Trucar a la funció en realitat porta a terme les accions especificades amb els paràmetres indicats. Per exemple, si defineixes la funció square, podries cridar-la amb square (5).

No es pot accedir a les variables definides dins d'una funció des de qualsevol lloc fora de la funció, perquè la variable es defineix només en l'àmbit de la funció. No obstant això, una funció pot accedir a totes les variables i funcions definides dins de l'àmbit en el qual està definida.

En altres paraules, una funció definida en l'àmbit global pot accedir a totes les variables definides en l'àmbit global. Una funció definida dins d'una altra funció també pot accedir a totes les variables definides en la seva funció principal i a qualsevol altra variable a la qual tingui accés la funció principal.


        // Las siguientes variables se definen en el ámbito global
                var num1 = 20,
                    num2 = 3,
                    name = 'Chamahk';

        // Esta función está definida en el ámbito global

                function multiply() {
                return num1 * num2;
                }

                multiply(); // Devuelve 60

        // Un ejemplo de función anidada

                function getScore() {
                var num1 = 2,
                    num2 = 3;

                function add() {
                    return name + ' anotó ' + (num1 + num2);
                }

                return add();
                }

        getScore(); // Devuelve "Chamahk anotó 5"

Let

La instrucción let declara una variable de alcance local con ámbito de bloque(block scope), la cual, opcionalmente, puede ser inicializada con algún valor.

let et permet declarar variables limitant el seu abast (scope) a el bloc, declaració, o expressió on s'està usando.a diferència de la paraula clau var la qual defineix una variable global o local en una funció sense importar l'àmbit de l'bloc. L'altra diferència entre var i let és que aquest últim s'inicialitza a un valor només quan un analitzador ho avalua (veure a baix).

A l'igual que const, let no crea propietats de l'objecte es declara globalment (en l'abast més alt).

Utilitza la paraula reservada let per definir variables dins d'un bloc.


        if (x > y) {
            let gamma = 12.7 + y;
            i = gamma * x;
              }

String

L'Objecte global String és un constructor per a strings, también conegudes com a Cadenes de Caràcters.

Les Strings són útils per a emmagatzemar tats que podin ser representades en forma de texte. D'algunes de les Operacions més emprades en strings són per a obtindre su llargada length, per a concatenar-les mitjançant a els Operadors de strings + i + =, per a comprovar l'existència o Localització de substrings amb el mètode indexOf (), o Bé per a extreure substrings amb el mètode substring ().

Number

L'Objecte de JavaScript Number és un Objecte embolcall que permet treballar amb valors numèrics. Un Objecte de tipo Number és creat Quan és crida el constructor Number ().

Els usos pricipals de l'objecte Number són: Si l'argument dóna a el constructor no és pot convertir en un nom AQUEST retorna NaN, si Number pot ser emprat per a realitzar Conversions de tipo sense necesitat de crear un nou objecte.

Exemples:

  • Number.parseFloat (): Fa la mateixa Funció que el mètode parseFloat de l'objecte global.

  • Number.parseInt (): Fa la mateixa Funció que el mètode parseInt de l'objecte global.

For

La sentència for crea un bucle que consta de tres expressions opcionals, embolcallades entre paràntesis i separades per punts i comes, seguits d'una sentència o un grup de sentències executades en el bucle.

  • Inicialització: Una expressió (incloent expressions d'assignació) o declaració de variables. Normalment s'usa per inicialitzar una variable al contador. Opcionalment aquesta expressió pot declarar noves variables amb la paraula clau var. Aquestes variables no sóc locals pel bucle, és a dir, es troben en el mateix àmbit en que es troba el bucle for. El resultat d'aquesta expressió es descarta.

  • Condició: Una expressió que s'avalua abans de cada iteració del bucle. Si aquesta expressió s'avalua true, s'executa la sentència. Aquesta prova condicional és opcional. Si s'omet, la condició sempre avalua true. Si l'expressió avalua a false, l'execució salta a la primera expressió que es trobi després del constructe for.

  • Expressió final: Una expressió per ser avaluada al final de cada iteració del bucle. Això passa abans de la següent avaluació de condició. Generalment s'usa per actualitzar o incrementar la variable contador

  • Sentència: Una sentència que s'executa mentre la condició avalui a true. Per executar múltiples sentències dins el bucle, utilitzeu una sentència block ({ ... }) per agrupar aquestes sentències. Per no executar cap sentència dins el bucle, utilitzeu una sentència buida (;).

Exemple:


        for (var i = 0; i < 9; i++) {
            console.log(i);
            // more statements
             }

While

la sentència while crea un bucle que executa una sentència especificada sempre que la condició s'avalui certa. La condició s'avalua abans d'executar la sentència.. 


        while (condició) {
            sentència
              }

Condició: Una expressió que s'avalua abans de cada passada del bucle. Si aquesta condició esdevé certa, s'executa la sentència. Quan la condició esdevé falsa, l'execució continua amb la sentència posterior al bucle while.

Sentència: Una sentència que s'executa sempre que la condició esdevingui certa. Per executar múltiples sentències dins un bucle, podeu fer ús de la sentència block ({ ... }) per agrupar aquestes sentències.


        var n = 0;
        var x = 0;

        while (n < 3) {
        n++;
        x += n;
                    }

Array

L'objecte de JavaScript Array és un objecte global que s'utilitza per a construir arrays; que són objectes d'al nivell que representen llistes.

Els arrays són objectes semblants a lliste el prototipus dels quals té mètodes que permeten realitzar operacions de travessa i mutació. Ni a longitud ni el tipus dels elements són fixos en els arrays de JavaScript. Com que el tamany d'un array pot canviar en qualsevol moment no es pot garantir la seva densitat. En general aquestes característiques són convenients; però si en algun cas aquestes característiques no fossin convenients es recomana considerar l'ús de arrays amb tipus.


        var fruites = ["Poma", "Pera"];

        console.log(fruites.length);
        // 2

For...of

La instrucció for...of crea un bucle que itera sobre objectes iterables (incloent Array, Map, Set, l'objecte arguments, etcètera), tot invocant un bloc de codi amb les instruccions a executar per a cada valor de la propietat.

El següent exemple mostra la diferència entre el bucle for...of i el bucle for...in. Mentre for...in itera sobre noms de propietats, for...of itera sobre els valors de les propietats:


        let arr = [3, 5, 7];
        arr.foo = "hola";

        for (let i in arr) {
        console.log(i); // mostra "0", "1", "2", "foo"
            }

        for (let i of arr) {
        console.log(i); // mostra "3", "5", "7"
            }

Debuger

El debuger et permet inspeccionar pas a pas el codi de JavaScript i examinar o modificar el seu estat ajudant a eliminar errors.

La icona d'Opcions del debuger, quan prems, presenta un menú d'opcions emergent que et deixa ajustar el comportament del debuguer.

Pausa en excepcions: Quan aquesta opció està marcada, l'execució de l'script es pausará automàticament cada vegada que es produeixi una excepció de JavaScript.

Mostra panells a l'inici: Quan aquesta opció està marcada, els dos panells de la barra lateral de l'depurador són visibles quan s'inicia el depurador per primera vegada; per defecte, no es veuen.

Mostra propietats ocultes: Si no marques aquesta opció, les propietats de JavaScript ocultes (és a dir, aquelles que són no-enumerables) no es mostren.

Mostra quadre de cerca de variables: Habilitar aquesta opció afegeix un quadre de cerca "Filtrar variables" a el panell de variables, que pugui filtrar la llista de variables que es mostra.
